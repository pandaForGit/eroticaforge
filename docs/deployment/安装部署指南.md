### 1. 硬件与系统要求

**最低配置（推荐）**：

- GPU：NVIDIA GTX 1660 6GB（推荐）或 GTX 1050 Ti 4GB（可用，但速度较慢）
- 系统内存：16GB RAM（推荐 32GB）
- 存储空间：至少 30GB 空闲（模型 + PostgreSQL 数据）
- 操作系统：Windows 10/11（推荐）或 Linux

**软件环境**：

- Java 21 或更高版本（JDK 21）
- Maven 3.8+
- NVIDIA CUDA 12.1 或 12.4（必须与驱动匹配，GPU 推理用）
- PostgreSQL 15+，并安装 **pgvector** 扩展
- Git
- **Ollama**（仅用于嵌入模型 **bge-m3**，默认端口 11434）

### 2. 安装步骤（按顺序执行）

#### 步骤 1：安装 NVIDIA 驱动和 CUDA

1. 打开 NVIDIA 官网下载最新 Game Ready Driver（推荐）
2. 下载并安装对应 CUDA Toolkit（建议 12.4）
3. 重启电脑后执行 `nvidia-smi`，确认显卡与 CUDA 信息

#### 步骤 2：安装 llama.cpp 并启动主生成服务

1. 从 https://github.com/ggerganov/llama.cpp 获取构建产物或自行编译
2. 准备 Qwen 4B 等 GGUF 模型文件
3. 启动 OpenAI 兼容 API（示例）：

```bash
./llama-server -m path/to/model-q4_k_m.gguf --host 127.0.0.1 --port 8081 --api-key dummy-key
```

确认 `http://localhost:8081/v1` 可访问。

#### 步骤 3：安装 Ollama 并拉取嵌入模型 bge-m3

1. 访问 https://ollama.com/download 安装 Ollama  
2. 拉取嵌入模型：

```bash
ollama pull bge-m3
```

3. 嵌入通过应用连接 `http://localhost:11434`，**不必**用 Ollama 跑主生成（主生成走步骤 2）。

#### 步骤 4：安装 PostgreSQL + pgvector

**Docker 示例**：

```bash
docker run -d --name pg-erotica \
  -e POSTGRES_USER=erotica \
  -e POSTGRES_PASSWORD=erotica \
  -e POSTGRES_DB=erotica \
  -p 5432:5432 \
  pgvector/pgvector:pg16
```

进入容器执行：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

表结构见 `docs/deployment/完整 PostgreSQL 与 pgvector 数据模型.md`。

#### 步骤 5：克隆并构建项目

```bash
git clone https://github.com/yourname/ai-writer.git
cd ai-writer
mvn clean install
```

#### 步骤 6：修改配置文件

编辑 `src/main/resources/application.yml`，核心示例：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/erotica
    username: erotica
    password: erotica

langchain4j:
  open-ai:
    base-url: http://localhost:8081/v1
    api-key: dummy-key
    chat-model:
      model-name: your-model-name
      temperature: 1.05
      max-tokens: 2048

  ollama:
    base-url: http://localhost:11434
    embedding:
      options:
        model: bge-m3

  # pgvector 存储的 dimension 与 bge-m3 一致（常见 1024，以实际 embed 为准）
```

#### 步骤 7：启动项目

```bash
mvn spring-boot:run
```

启动成功后访问主界面与 API（若已配置 Swagger）。

### 3. 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| llama-server 无响应 | 端口或路径错误 | 检查 `base-url` 与模型路径 |
| Ollama 嵌入失败 | 未拉取 bge-m3 | 执行 `ollama pull bge-m3` |
| pgvector 插入失败 | 维度不匹配 | 对齐 `vector(N)` 与 bge-m3 输出维度 |
| 显存不足 | 模型过大 | 使用 Q4_K_M 或更小上下文 |

### 4. 启动后检查清单

- [ ] `nvidia-smi` 在生成时有合理显存占用（若使用 GPU）
- [ ] `llama-server` 与 Ollama（embedding）均可连通
- [ ] PostgreSQL 已创建扩展 `vector`，应用可建表/读写
- [ ] 上传测试文档后，可在库中查到 `erotica_rag_chunks` 或等价表数据

### 5. 一键启动脚本（可选）

按需先启动 Docker 中的 PostgreSQL，再启动 `llama-server`，最后 `mvn spring-boot:run`。
