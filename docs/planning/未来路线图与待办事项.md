**未来路线图与待办事项.md**

**文档版本**：1.0  
**更新日期**：2026-03-24  
**项目名称**：EroticaForge - 本地 NSFW 角色扮演小说生成工具

### 1. 产品愿景
打造一款**极致本地化、记忆超强、NSFW 完全自由**的长篇角色扮演小说创作工具，最终目标是让用户能够像使用 genrati.xyz + xxss.ai 的结合体一样顺畅创作，但全部运行在自己的电脑上，零审查、零数据泄露。

### 2. 版本路线图（Roadmap）

#### v1.0 - MVP（已基本完成 / 进行中）
**目标**：可实际用于日常长篇 NSFW 小说创作  
**核心功能**：
- 单模型流式生成
- PostgreSQL（pgvector）持久化 RAG
- StoryState 故事状态机
- 基础 Web UI（Thymeleaf + HTMX）
- 章节管理与导出
- 防重复与防 OOC 基础机制

**完成标准**：能稳定连写 20 章以上不明显崩坏

**当前状态**：核心功能已实现，文档体系完善

#### v1.1 - 多模型增强版（优先级最高，建议接下来重点开发）
**计划时间**：2026 年 4 月  
**主要特性**：
- 三步多模型协作链（嵌入 → 4B 总结 → 9B 生成）
- 自动 StoryState 更新与合并
- 更强的防死循环机制（动态 penalty + n-gram 屏蔽）
- Lorebook 管理界面
- 生成质量评分与手动反馈
- 性能监控仪表盘（显存、token/s、生成耗时）

#### v1.2 - 体验优化版（2026 年 5 月）
**主要特性**：
- Vaadin 或纯 React 前端（可选，用户可二选一）
- 更美观的写作界面（类似 SillyTavern 的卡片式布局）
- 一键导入/导出整本小说（Markdown / EPUB）
- 角色卡可视化编辑器
- 批量文档上传与智能分类
- 暗黑模式 + 自定义主题

#### v2.0 - 多模态增强版（2026 年 6-7 月）
**主要特性**：
- 支持**图生文**（Image-to-Text）：上传 NSFW 图片自动生成描述或续写剧情
  - 集成 JoyCaption / Qwen2.5-VL uncensored
- 基于图片的场景生成（看图写文）
- 简单图像生成功能（调用本地 SDXL / Flux，轻量版）
- 语音输入提示（可选）

#### v2.1 - 高级创作工具（2026 年 8 月以后）
- 自动剧情大纲生成器
- 章节标题智能生成
- 风格迁移（可切换不同色情写作风格）
- 多故事并行管理 + 故事分支（What-If 模式）
- 导出为专业小说格式（EPUB + 目录 + 封面）

#### v3.0 - 社区与扩展版（长期目标）
- Docker 一键部署包
- 支持多种后端（llama.cpp、vLLM、LM Studio）
- 插件系统（自定义 Prompt、自定义后处理）
- 可选云端同步（仅元数据，用户可关闭）

### 4. 优先级排序原则
1. 一致性与记忆（长篇写作核心）
2. NSFW 服从度与描写质量
3. 用户体验（流式输出、界面友好度）
4. 性能与硬件适配（1660 6GB 友好）
5. 扩展性

### 5. 风险与注意事项
- 多模型链会增加延迟，需做好用户提示和开关控制
- 图生文模型对显存要求较高，需提供轻量选项
- 所有新增功能必须同步更新测试用例和用户手册
