# 人物卡库 + 故事快照 — 实施报告

**日期**：2026-04-03  
**范围**：按计划落地「JSONL 导入库表 → 创建故事可选克隆快照 → 快照 CRUD/排序 → 生成 Prompt 注入 `{{characterProfiles}}`」及前端最小闭环。

---

## 1. 数据库

- 新增脚本：`sql/002_character_library_and_snapshots.sql`  
  - `erotica_character_library`：`UNIQUE(content_sha256, role_index)`，JSONB `payload`。  
  - `erotica_story_character_snapshots`：`story_id` 级联删除，`cloned_from_library_id` 对库表 `ON DELETE SET NULL`。

**部署注意**：在已有库上手工执行该脚本（与 `001_init_pgvector.sql` 独立）。

---

## 2. 后端（Spring Boot）

| 模块 | 说明 |
|------|------|
| 实体 / Mapper / 仓储 | `CharacterLibraryEntity`、`StoryCharacterSnapshotEntity` 及对应 Mapper、Repository |
| 导入 | `CharacterCardsJsonlImporter` + `CharacterCardsJsonlImportApplicationRunner`；配置 `erotica.character-cards-import`（`application.yml`） |
| 业务 | `StoryCharacterSnapshotService`（快照 CRUD、排序、创建故事批量克隆）、`CharacterProfilePromptFormatter` |
| 写故事 | `StoryWriteFacade.insertStoryWithInitialState(Story, List<String> libraryCharacterIds)` 事务内写故事 + 初始状态 + 快照 |
| API | `CharacterLibraryController`（`GET /api/character-library`）、`StoryCharacterSnapshotController`（快照 REST）、`CreateStoryRequest` 增加 `libraryCharacterIds` |
| 生成 | `NovelGenerationService` 拉快照文本；`PromptComposer.buildFullPrompt` 增加 `characterProfiles` 参数；`PromptProperties` 默认模板含 `{{characterProfiles}}` 区块 |
| 入口 | `EroticaForgeApplication` 注册 `CharacterCardImportProperties` |

**配置示例**：`erotica.character-cards-import.enable=true` 且 `jsonl-path` 指向 `dataAnalysisModule/out/character_cards.jsonl`（或 `CHARACTER_CARDS_JSONL` 环境变量）。

**说明**：新建实体未使用 Lombok `@Data`（干净编译环境下未生成 accessor 会导致失败），已改为手写 getter/setter。

---

## 3. 前端（Vite + React）

- `StoryCreatePage`：人物卡库多选（顺序即快照顺序），提交 `libraryCharacterIds`。  
- `StoryCharacterSnapshotsPanel`（故事详情）：列表、从库添加、空白快照、JSON 编辑、删除、上移/下移（调用 `PUT .../order`）。  
- API：`api/characterLibrary.ts`、`api/storyCharacterSnapshots.ts` 与对应类型。

## 6. 已知限制 / 后续可做

- 人物卡导入目前仅 **启动 Runner**（与参考语料导入同模式）；未提供独立管理 REST。  
- 删除库卡不会删除已克隆快照（`ON DELETE SET NULL`）。  
- P2：人设向量检索、库卡在线编辑/合并等仍按产品文档延后。

## 7. 建议你本地验证的步骤

1. PostgreSQL 执行 `sql/002_character_library_and_snapshots.sql`。  
2. 设置 `erotica.character-cards-import.enable=true` 与正确 `jsonl-path`，启动后端，查看日志导入统计。  
3. 前端新建故事勾选人物 → 详情页改快照 → 生成一页，核对日志或调试中完整 Prompt 是否含「本故事人物设定」块。
