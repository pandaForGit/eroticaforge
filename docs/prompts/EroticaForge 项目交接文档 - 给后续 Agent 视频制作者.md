### 1. 项目核心理念（最重要，请先理解）

用户真实需求是：
- 先制作**人物卡**和**世界观设定**
- 通过**描述行动 + 与角色对话**的方式进行互动

- 强调**长篇一致性**（不 OOC、不忘设定、不重复描写）

**核心体验目标**：像 SillyTavern + NovelAI 的结合体，但全部跑在本地。

### 3. 项目核心功能流程（简要版）

1. 用户创建故事 → 上传人物卡 + 世界观
2. 用户输入行动/对话（可使用 OOC 指令）
3. 系统流程：
   - RAG 检索相关记忆
   - 读取 StoryState（当前剧情摘要 + 人物状态）
   - 构建完整 Prompt
   - （可选）4B 总结模型先压缩内容
   - 9B 主模型生成小说形式回复（旁白 + 对话气泡 + 内心独白）
4. 用户可手动编辑生成内容 → 保存为章节 → 继续互动

### 4. 重要设计点（请务必理解）

- **不是简单续写小说**，而是**互动角色扮演 + 小说叙述**的混合模式
- 重点解决长篇问题：RAG + StoryState 双重记忆机制
- 强烈依赖高质量**人物卡**（这是提升一致性的关键）
- 支持多模型协作（单模型快，多模型一致性更高）
- 生成内容可实时编辑 + 保存章节
