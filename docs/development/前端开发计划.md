## 1. 目标与约束

### 1.1 本计划要解决什么

在**后端已提供 REST + SSE**（见阶段 5：`/api/stories/**`、`/api/health`、`/api/lorebook` 等）的前提下，给出**可执行的前端实现顺序**：页面与模块划分、技术选型建议、与接口的对应关系、验收标准，便于单人或 Agent 按阶段推进。

### 1.2 前提假设

- **Base URL**：默认 `http://localhost:8090/api`（以运行中的 Spring Boot `server.port` 为准，与 `application.yml` 一致；勿写死文档中的 8080 示例端口）。
- **认证**：当前无 Token；所有请求直连即可。
- **内容类型**：生成内容为成人向 NSFW 文本；UI 需考虑**本地单机使用**场景（非公网多用户）。

### 1.3 首版不强制完成的内容（可标为后续迭代）

- 账号体系、权限、云端同步  
- 富文本协同编辑、离线 PWA（可选）  
- 与数据分析模块（`dataAnalysisModule`）的联调界面  
- 完整无障碍与国际化（i18n）

## 2. 技术选型建议（可调整，但需在仓库 README 或本文档回写）

| 维度 | 建议 | 说明 |
|------|------|------|
| 框架 | **Vite + React** 或 **Vite + Vue 3** | 生态成熟，易接 SSE；团队熟悉者优先 |
| 语言 | **TypeScript** | 与 OpenAPI/手写类型对齐，减少字段拼写错误 |
| HTTP | **fetch** 或 **axios** | 统一封装 Base URL、错误体解析（`code/message/error` 与 `ApiResponse`） |
| 流式生成 | **EventSource** 或 **fetch + ReadableStream** | 对接 `POST .../generate/stream` 的 `text/event-stream`；注意 Spring 返回的 `data: {...}` 解析 |
| 状态 | **TanStack Query** + 轻量全局 store（可选 Zustand） | 故事列表、章节、state 以服务端为准，缓存与失效策略清晰 |
| 样式 | **CSS Modules / Tailwind** 任选 | 保持本地工具风格简洁即可 |

**说明**：若选用 React，SSE 若需 `POST` + body，浏览器原生 `EventSource` 仅支持 GET，需使用 **`fetch` 流式读取** 或后端为开发环境单独提供 GET 代理（当前后端为 POST SSE，**推荐 fetch + stream**）。

## 3. 总体阶段划分

```text
阶段 0 环境与工程骨架
    ↓
阶段 1 API 客户端与错误处理（对齐 ApiResponse / ApiErrorResponse）
    ↓
阶段 2 故事：列表 / 创建 / 详情 / 删除
    ↓
阶段 3 文档：上传（multipart）与文档列表
    ↓
阶段 4 生成：SSE 流式展示 + 非流式调试入口（可选）
    ↓
阶段 5 阅读：章节列表与单章正文
    ↓
阶段 6 状态与 Lorebook：StoryState 展示与调试编辑、Lorebook 简易管理
    ↓
阶段 7 联调与健康：/api/health 仪表盘、文案与基础 UX
```

## 4. 分阶段任务与验收

### 阶段 0：环境与工程骨架

| 项 | 内容 |
|----|------|
| **任务** | 初始化前端仓库或 monorepo 子目录；配置 `VITE_API_BASE`（或等价）指向 `http://localhost:8090/api`；配置代理避免 CORS（开发态）。 |
| **产出** | 可 `npm run dev` 打开空白壳页面；环境变量示例 `.env.example`。 |
| **验收** | 浏览器能访问首页；控制台无致命报错。 |

### 阶段 1：API 客户端与错误处理

| 项 | 内容 |
|----|------|
| **任务** | 封装请求：`GET/POST/PUT/DELETE`；成功解析 `data`；失败解析 `code/message/error` 并统一 Toast/弹窗。 |
| **参考** | `docs/api/API 接口定义.md` 第 4 节。 |
| **产出** | `api/client.ts`（或同级）+ 各资源模块占位。 |
| **验收** | 故意请求不存在的 `storyId`，界面展示后端返回的错误信息而非白屏。 |

### 阶段 2：故事管理

| 项 | 内容 |
|----|------|
| **任务** | 对接 `POST /stories`、`GET /stories`、`GET /stories/{id}`、`DELETE /stories/{id}`；列表展示标题、标签、章节数、更新时间。 |
| **产出** | 故事列表页、创建表单、详情页骨架。 |
| **验收** | 完成「创建 → 列表可见 → 进入详情 → 删除」闭环。 |

### 阶段 3：文档上传与列表

| 项 | 内容 |
|----|------|
| **任务** | `POST /stories/{id}/documents`（`file` + 可选 `metadata` JSON 字符串）；`GET .../documents` 列表展示 `docId、fileName、chunkCount`。 |
| **产出** | 故事详情内「文档」Tab 或独立页。 |
| **验收** | 上传 UTF-8 `.txt` 成功；列表显示 `chunkCount`；超大文件触发 413 时有提示。 |

### 阶段 4：生成（核心）

| 项 | 内容 |
|----|------|
| **任务** | 对接 `POST /stories/{id}/generate/stream`：**流式追加正文**；监听结束帧 `done: true` 与 `chapterId`；可选：`POST /generate` 非流式用于排错。 |
| **注意** | `useMultiModel: true` 时后端返回 501，前端应禁用开关或提示「未实现」。 |
| **产出** | 生成页：输入 `prompt`、开始按钮、中断（AbortController）、生成区滚动到底。 |
| **验收** | 在 llama-server 与 Ollama 正常时，能看到 token 级流式输出；结束后能拿到 `chapterId` 并提示成功。 |

### 阶段 5：章节阅读

| 项 | 内容 |
|----|------|
| **任务** | `GET .../chapters` 侧边或列表；`GET .../chapters/{chapterId}` 渲染 Markdown（若正文含轻量 Markdown）。 |
| **产出** | 章节导航 + 阅读视图。 |
| **验收** | 生成新章后刷新列表可见；点开可读全文。 |

### 阶段 6：StoryState 与 Lorebook

| 项 | 内容 |
|----|------|
| **任务** | `GET/PUT .../state`：展示 `version、summary、人物状态、facts`；PUT 时携带正确 `version` 处理 409 提示刷新。`GET/POST /lorebook` 简易表格增删查（知悉当前后端为内存实现，重启清空）。 |
| **产出** | 调试向「状态 JSON」面板 + Lorebook 页。 |
| **验收** | PUT state 冲突时有明确文案；Lorebook 新增后能在下次生成逻辑中生效（关键词命中由后端实现）。 |

### 阶段 7：健康检查与体验收尾

| 项 | 内容 |
|----|------|
| **任务** | 调用 `GET /health`，展示 `database、llm、embedding` 状态，便于用户自检环境。 |
| **产出** | 设置页或页脚「服务状态」。 |
| **验收** | 关数据库或 llama 时 UI 能显示 `down` 类状态，并链到 `docs/deployment/安装部署指南.md`（或本地 README）。 |

## 5. 与后端接口映射速查

| 前端模块 | 方法 | 路径 |
|----------|------|------|
| 健康 | GET | `/health` |
| 故事 | POST/GET/DELETE | `/stories`、`/stories/{storyId}` |
| 文档 | POST/GET | `/stories/{storyId}/documents` |
| 生成 | POST | `/stories/{storyId}/generate/stream`、`/generate` |
| 章节 | GET | `/stories/{storyId}/chapters`、`.../chapters/{chapterId}` |
| 状态 | GET/PUT | `/stories/{storyId}/state` |
| Lorebook | GET/POST | `/lorebook` |

（完整字段与示例见 `docs/api/API 接口定义.md`。）
