### 1. 模型总体策略

本项目采用 **主生成（llama.cpp）+ 嵌入（Ollama bge-m3）+ 可选总结模型** 的组合：

| 模型类型 | 推荐 | 说明 |
|----------|------|------|
| **主生成模型** | Qwen 4B 量级 GGUF，经 **llama.cpp** `llama-server` 提供 OpenAI 兼容 API | 见 `docs/architecture/本地 NSFW 小说制作搭建计划--java.md` |
| **总结模型**（多步链可选） | 可与主模型同实例顺序调用，或单独 4B/7B | 低温、输出结构化 JSON |
| **嵌入模型** | **bge-m3**（通过 **Ollama** 运行，供 RAG 写入 **PostgreSQL + pgvector**） | 向量维度以实际 embed 为准，一般为 **1024**（部署后校验） |

**推荐使用原则**：

- 日常写作：单主模型 + 强 Prompt。  
- 长篇高一致性：三步链（检索 → 总结 → 生成），嵌入统一 **bge-m3**。  
- 存储：**向量与业务数据在 PostgreSQL（pgvector）**，不再使用 Redis 作为向量库。

### 2. 模型准备命令

```bash
# 嵌入模型（RAG 必须）— 通过 Ollama 拉取 bge-m3
ollama pull bge-m3

# 若总结/辅助仍走 Ollama（可选，与主路径解耦时）
ollama pull qwen3.5:4b
```

主生成模型为 **GGUF 本地文件**，由 llama.cpp 加载，不以 `ollama pull` 为准。

### 3. application.yml 配置示例（LangChain4j）

**要点**：`open-ai` 对接 llama-server；`ollama` 仅用于 **embedding（bge-m3）**。

```yaml
langchain4j:
  open-ai:
    base-url: http://localhost:8081/v1
    api-key: dummy-key
    chat-model:
      model-name: qwen-4b-uncensored
      temperature: 1.05
      max-tokens: 2048

  ollama:
    base-url: http://localhost:11434
    embedding:
      options:
        model: bge-m3

  # PostgreSQL + pgvector 在 Spring 数据源与 LangChain4j PgVector 适配器中配置，
  # dimension 须与 bge-m3 输出维度一致（常见为 1024，请用一次 embed 确认）。
```

若你使用的 LangChain4j 版本将 `embedding` 配在 `ollama` 下略有差异，以保持 **`base-url` + `model: bge-m3`** 为准，并查阅当前版本的 `langchain4j-ollama` 文档。

### 4. 与数据层的关系

- **bge-m3** 产出向量 → 写入 `docs/deployment/完整 PostgreSQL 与 pgvector 数据模型.md` 中的 `erotica_rag_chunks`。  
- 勿混用旧版 768 维（nomic）配置；迁移后索引需按新维度重建。

### 5. 常见问题

| 问题 | 处理 |
|------|------|
| 嵌入维度与 pgvector 列不一致 | 以 `embed("test").length` 为准，修改 `vector(N)` 与 LangChain4j `dimension` |
| Ollama 仅跑 embedding、llama 跑生成 | 属正常拆分；健康检查应对 **两处** 探测 |
