你的项目是一个**本地 NSFW 小说/角色扮演生成工具**，核心需求包括：

- 全本地运行（**llama.cpp** 主生成 GGUF + **Ollama** 嵌入 **bge-m3**）
- 持久化 RAG（**PostgreSQL + pgvector**，剧情/人物卡/章节总结永不丢失）
- 纯 Java 技术栈（Spring Boot + LangChain4j）
- 支持长上下文、实时流式输出、避免死循环/重复
- 交互式写作（类似 SillyTavern 的 Story 模式：续写、角色卡、Lorebook 触发、上传文档）

下面给出**推荐的项目架构设计**（2026 年视角，基于 LangChain4j + Spring Boot 的最佳实践），分层清晰、可扩展、易维护。

### 关键设计原则 & 推荐实践

1. **分层职责清晰**（避免 God Class）
   - domain：纯业务概念，无 Spring 注解
   - application：业务用例协调（service 层调用 domain + infrastructure）
   - infrastructure：外部依赖适配（PostgreSQL、llama.cpp、Ollama 嵌入、文件系统）
   - presentation：暴露接口（REST + WebSocket/SSE）

2. **RAG 流程拆分**（持久化 + 可观测）
   - Ingestion：用户上传 .txt / .pdf → Document → split → embed（**bge-m3**）→ **pgvector EmbeddingStore.addAll()**
   - Retrieval：query → embed → 向量相似度搜索 → 过滤（相似度阈值 0.75+）→ 组装 context
   - Generation：AiServices 或手动链：system prompt + retrieved context + user input → StreamingChatResponse

3. **故事一致性 & 防循环核心**
   - 每生成一段后，调用 StoryStateService 做**自动总结**（用小模型或规则提取关键事件/状态）
   - 把最新 StoryState 塞进 prompt（代替全历史）
   - 系统提示强制 anti-loop（见之前建议）
   - 参数全局统一：temperature 1.0–1.1, repeat_penalty 1.15+, top_p 0.92

4. **持久化策略**
   - **向量数据**：PostgreSQL **pgvector**（见 `完整 PostgreSQL 与 pgvector 数据模型.md`）
   - **元数据 / 故事树**：同库表 + JSONB
   - **文件备份**：novel-docs/ 目录 + 定期 `pg_dump`

5. **接口设计建议**
   - REST（简单测试）：
     - POST /api/novel/upload → 上传文档 → 触发 ingestion
     - POST /api/novel/generate → {prompt, storyId} → 返回完整一段或流式
   - WebSocket / SSE（实时写作感）：
     - /ws/story/{storyId} → 订阅流式输出
     - 支持 “续写下一段”、“OOC 指令”、“切换视角”等

6. **可扩展点**
   - 多模型支持：通过配置切换 chat / summarizer
   - 多用户 / 多故事：`story_id` 作为隔离键（表列 + RAG metadata）
   - 日志 / 监控：Micrometer + Prometheus 监控推理延迟、token/s
   - 测试：Mock llama-server + Testcontainers PostgreSQL

### 推荐启动顺序 & 最小 MVP

1. NovelApplication + config 包（DataSource + Ollama embedding + llama OpenAI client）
2. RagIngestionService + RagRetrievalService（测试上传 & 检索）
3. NovelGenerationService（AiServices 方式最简单）
4. NovelController（先 REST，后加 SSE）
5. 前端（Thymeleaf + HTMX 或纯 JS WebSocket）

这个结构在 LangChain4j + Spring Boot 的 RAG 项目中很常见，适合从 MVP 逐步迭代到完整 NSFW 写作工具（加角色卡管理、Lorebook 关键词匹配、图像生成接口等）。
