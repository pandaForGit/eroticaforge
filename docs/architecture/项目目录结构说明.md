# 项目目录结构说明

**文档版本**：1.1  
**更新日期**：2026-03-28  
**项目名称**：EroticaForge（本地 NSFW 角色扮演小说生成工具）

本文描述仓库**当前**布局，与 `com.eroticaforge` 主工程及 `dataAnalysisModule` 独立模块一致。

---

## 1. 总体目录结构

```text
ai-writer/
├── docs/                              # 项目文档（本文件所在体系）
├── src/main/java/com/eroticaforge/    # 主后端 Spring Boot 源码
├── src/main/resources/
│   ├── application.yml                # 主配置（端口默认 8090、DB、LangChain4j、erotica.*）
│   └── application-local.yml        # 可选本地覆盖（optional import）
├── src/test/java/                     # 主工程测试
├── dataAnalysisModule/                # 独立 Maven 工程：语料分析 / 分类 / 人物卡批处理
├── frontend/                          # 前端（若使用）
├── sql/                               # PostgreSQL 脚本
├── pom.xml                            # 主工程 Maven 父/聚合配置
└── README.md
```

### 各目录职责

| 包 | 职责 |
|----|------|
| `config` | Spring Bean、外部化配置绑定 |
| `domain` | 领域对象与异常；避免依赖 Web 与持久化细节 |
| `application` | 用例编排：生成、RAG、状态、接入校验 |
| `infrastructure` | 数据库访问、与 LangChain4j 存储等基础设施 |
| `presentation` | HTTP API、统一异常处理 |
| `utils` | 无状态工具与模板 |

### `src/main/resources`

- `application.yml`：数据源、`langchain4j.open-ai`（llama-server OpenAI 兼容）、`langchain4j.ollama`（bge-m3）、`erotica.rag` / `erotica.generation` / `erotica.pgvector` 等。  
- 默认 **不与 llama-server 抢 8080**：`server.port` 常为 `8090`（见配置文件注释）。

## 3. 数据分析模块 `dataAnalysisModule/`

独立 artifact（如 `eroticaforge-data-analysis`），包名 `com.eroticaforge.dataanalysis`，与主工程**无源码依赖**；共用 llama-server / 环境变量约定。详见模块内 `pom.xml`、`application.yml` 及 `dataAnalysisModule/数据分析模块开发计划.md`。

典型包：

- `corpus/` — 语料清洗、分类（如 `NovelClassifierService`、`CorpusSanitizer`）  
- `charactercard/` — 人物卡提取与批处理 Runner  
- `support/` — 直连 Chat Completions 客户端、JSON 抽取与修复等

## 4. 核心文件快速索引

| 路径 | 作用 |
|------|------|
| `EroticaForgeApplication.java` | 启动入口 |
| `NovelGenerationService.java` | 构建 Prompt、调用流式/阻塞 ChatModel |
| `PostGenerationService.java` | 生成后落库章节、更新 StoryState、可选 RAG 再索引 |
| `StoryStateService.java` | 读取/更新故事状态（含乐观锁） |
| `RagIngestionService.java` | 文档切块、嵌入、写入向量存储与 `erotica_documents` |
| `RagRetrievalService.java` | 按 `story_id`（及可选参考库）检索并格式化为上下文 |
| `CorpusJsonlReferenceImporter.java` | 将分类 JSONL 与语料根目录对齐，写入参考库 story |
| `PromptProperties` + `PromptComposer` | `erotica.prompt.*` 可配置提示词与拼装 |
| `application.yml` | 运行时配置总入口 |

---

## 5. 包依赖原则

- **domain** 不依赖 application / infrastructure / presentation  
- **application** 可依赖 domain；通过接口或门面访问 infrastructure  
- **infrastructure** 可依赖 domain；不依赖 presentation  
- **presentation** 依赖 application 提供的服务，避免在 Controller 中堆业务逻辑  
- 禁止循环依赖

## 6. 新增代码放置建议

| 类型 | 位置 |
|------|------|
| 新领域概念或异常 | `domain/` |
| 新用例 / 流程 | `application/service/` |
| 新 REST 契约 | `application/dto/api/` + `presentation/controller/` |
| 新表访问 | `infrastructure/persistence/` |
| 新集成配置 | `config/` |
| 设计说明 | `docs/`（并在 `docs/README.md` 中挂链） |
