### 2. 各目录详细说明

#### docs/
存放所有项目文档，是开发和维护的重要参考资料。

#### src/main/java/com/eroticaforge/

- **config/**：Spring Boot 配置类（PostgreSQL、Ollama embedding、llama.cpp、LangChain4j 等 Bean 定义）
- **domain/**：核心业务实体类（不包含任何框架注解，保持纯净）
- **application/**：业务逻辑核心
  - `service/`：所有核心 Service（最重要部分）
  - `dto/`：接口请求和响应对象
- **infrastructure/**：外部系统适配器（PostgreSQL/pgvector、Ollama 嵌入、llama.cpp 调用等）
- **presentation/**：控制器和 WebSocket 处理（对外暴露接口）
- **utils/**：通用工具类（表名常量、Prompt 模板、防循环工具等）

#### src/main/resources/

- `application.yml`：主要配置文件（端口、数据库、llama-server、Ollama embedding 等）
- `static/`：前端静态文件（JS、CSS）
- `templates/`：Thymeleaf HTML 页面

#### 根目录重要文件

- `pom.xml`：Maven 依赖管理
- Ollama 模型缓存：由 `ollama` 管理（如 `bge-m3`）；主生成 GGUF 放在本地任意目录并由 llama-server 指定
- `start.bat`：Windows 一键启动脚本
- `README.md`：项目入口说明（简要版用户手册）
- `CHANGELOG.md`：版本变更记录

### 3. 核心文件快速索引

| 文件路径 | 作用 |
|---------|------|
| `NovelApplication.java` | Spring Boot 主启动类 |
| `NovelGenerationService.java` | 小说生成核心逻辑（最重要） |
| `StoryStateService.java` | 故事状态维护（记忆一致性关键） |
| `RagIngestionService.java` | 文档上传与 RAG 索引 |
| `DbTableNames.java` | 与 PostgreSQL 表名、列约定一致（可选） |
| `PromptTemplates.java` | 存放所有 Prompt 模板 |
| `application.yml` | 系统配置入口 |
| `StoryController.java` | REST API 控制器 |

### 4. 包依赖原则（推荐遵守）

- **domain** 不依赖任何其他包
- **application** 可依赖 domain 和 infrastructure
- **infrastructure** 可依赖 domain
- **presentation** 只依赖 application 服务层
- 禁止循环依赖

### 5. 新增文件建议

当你新增功能时，请按以下规则放置：
- 新实体 → `domain/`
- 新业务逻辑 → `application/service/`
- 新外部适配器 → `infrastructure/`
- 新接口 → `presentation/controller/`
- 新工具类 → `utils/`
- 新文档 → `docs/`
