### 1. 隐私承诺
**EroticaForge 的核心设计理念就是“完全本地、零数据泄露”**。

我们郑重承诺：
- **所有内容 100% 在您的电脑本地运行**，绝不上传任何小说内容、人物设定、生成记录或图片到云端。
- **不会收集任何用户数据**，包括使用习惯、生成内容、硬件信息等。
- **不会内置任何遥测、分析或联网功能**（除非您手动启用 Ollama 的可选更新检查）。

您创作的所有 NSFW 小说、角色卡、剧情设定，都只属于您自己。

### 2. 数据存储位置与保护

| 数据类型           | 存储位置                                   | 是否加密 | 说明 |
|--------------------|--------------------------------------------|----------|------|
| 小说章节与向量     | PostgreSQL（pgvector + 业务表）            | 可选     | 本地数据库数据目录或 Docker 卷 |
| 人物卡 / 大纲      | PostgreSQL + 原始文档文件夹        | 可选     | 默认存于 `docs/` 目录 |
| 故事状态（StoryState） | PostgreSQL（JSONB）                                 | 可选     | 包含剧情摘要和人物状态 |
| 生成 Prompt        | 仅内存中临时存在                           | -        | 生成完成后立即释放 |
| 模型文件           | Ollama 缓存（`~/.ollama/models`，嵌入 bge-m3）+ 本地 GGUF（llama.cpp 主模型）  | -        | 均本地 |

**重要提醒**：
- PostgreSQL 数据默认保存在本地磁盘（Docker 映射卷或 `PGDATA`）。
- 建议定期备份：`pg_dump` 或复制数据卷。
- 若使用 Docker 运行 Postgres，建议使用持久化卷（volume）。

### 3. 网络连接情况

- **默认状态**：**完全离线运行**，无需连接互联网。
- 唯一可能联网的情况：
  - 首次安装 Ollama 时下载模型
  - Ollama 检查更新（可通过 Ollama 设置关闭）
  - Docker 拉取 PostgreSQL 镜像（仅首次）

生成小说过程中**不会产生任何网络请求**。

### 4. NSFW 内容安全说明

- 本工具**不包含任何内容审查机制**，生成的 NSFW 内容完全由您输入的 Prompt 和模型决定。
- 所有生成内容仅存储在您的本地设备上。
- **我们强烈建议**：
  - 使用单独的用户账号运行本项目
  - 定期清理不想要的生成记录
  - 将项目目录设置为仅自己可访问

### 5. 安全最佳实践建议

1. **使用专用文件夹**  
   将整个项目放在 `D:\AI\NSFW-Writer\` 等非公开目录下。

2. **PostgreSQL 访问控制**（可选加强）  
   为数据库设置强密码，限制仅本机监听（`listen_addresses`）。

3. **定期清理**  
   - 生成大量敏感内容后，可通过 SQL 或管理工具删除旧章节与向量行。
   - 或按备份策略恢复/裁剪数据集。

4. **备份策略**  
   - 每周备份一次 `docs/` 目录 + `pg_dump` 导出。
   - 可使用简单脚本自动导出所有故事为 Markdown。

5. **卸载清理**  
   如果您不再使用本项目，请同时删除：
   - 项目文件夹
   - Ollama 模型缓存（`~/.ollama/models`）
   - PostgreSQL 数据目录
